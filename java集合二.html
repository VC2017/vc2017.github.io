Connection set

Set是Collection子接口，Set所包含de 方法和Collection基本一样，具体可参考jdk

set

无法记住添加的顺序，不允许包含重复元素

判断两个对象是否相等用equals，而不是用==

常用子类
    HashSet：散列存放
    TreeSet：有序存放

HashSet

特点

1 采用hash算法存储数据具有良好的存储和查找功能
2 不记录添加顺序
3 线程不安全的，多个线程访问一个HashSet要使用同步代码（synchronized ）
4 HashSet集合元素值允许是null（最多一个）

hash算法

1 可以通过一个对象快速找到另一个对象
2 其算法价值在于可以快速查询

算法简述：

        当从HashSet中访问元素时,HashSet先计算该元素的hashCode，然后直接找到该元素

TreeSet

对元素进行自然排序，或者是根据创建set时提供的Comparator 进行排序
排序规则：参与排序的元素必须是同一类型的

comparator():返回当前Set使⽤用的Comparator,若返 回null,表示以自然顺序排序。
Object ﬁrst()：返回set中当前第一个(低)元素。
Object last(): 返回此 set 中当前后一个(高)元素。 
SortedSet subSet(Object  fromElement, E toElement): 返回此 set 的部子集，其元素从 fromElement（包括）到 toElement（不包括）。 
SortedSet headSet(Object  toElement):返回此 set 的部分⼦子集， 其元素严格小于 toElement。
SortedSet tailSet(Object  fromElement) :返回此 set 的部分子集，其元素大于等于 fromElement。

自然排序：实现comparable接口 覆写comparableTo方法
定制排序：实现comparator将自己写的Comparator传入主函数
